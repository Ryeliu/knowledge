# 文件归档 Skill

## 输入

扫描 `~/worklab/Sidejob/knowledge/inbox/files/` 中的所有文件。

如果用户通过 `$ARGUMENTS` 指定了公司名，则将所有文件归入该公司。否则自动识别。

## 执行流程

### Step 1：扫描 inbox/files

列出 `inbox/files/` 中所有文件（排除 `.DS_Store`）。如果为空，告知用户没有待归档文件并结束。

### Step 2：识别与分组

**对每个文件：**

1. **根据文件名推断公司**：
   - 文件名中包含公司名/品牌名/产品名 → 归入该公司
   - 例如：`UniAction产品介绍.pdf` → 言生行(UniAction)
   - 例如：`润信AI方案.pdf` → 润信科技

2. **如果文件名无法判断**，读取文件内容前几页/段落推断：
   - PDF：用 `pdftotext` 提取前2页文字
   - docx/doc：用 python3 docx 库或 textutil 提取文字
   - md/txt：直接读取前100行
   - 其他格式（xlsx/xmind/ppt等）：根据文件名和上下文推断

3. **如果仍无法判断**，向用户提问确认归属

4. 将所有文件按公司分组，向用户确认分组是否正确后再执行

### Step 3：格式转换

- `.docx` → 提取文字，写成 `.md` 文件（标题用文件名），删除原 docx
- `.doc` → 用 `textutil -convert txt` 提取，写成 `.md`，删除原 doc
- `.pdf` / `.xlsx` / `.xmind` / 图片等 → 保持原格式不转换

### Step 4：归档文件

**对每个公司组：**

1. 检查 `companies/` 下是否已有该公司目录
   - **已有**：将文件移入该公司目录
   - **新公司**：创建 `companies/公司名/` 目录

2. 移动文件到目标目录

### Step 5：更新 README.md

**对每个涉及的公司：**

1. 如果是**新公司**（无 README.md）：
   - 读取归档的文件内容，提取公司信息
   - 按知识库 CLAUDE.md 中的公司档案格式创建 README.md
   - 包含：全称、核心产品、技术特点、与我的关系、关键联系人、原始资料索引

2. 如果是**已有公司**（有 README.md）：
   - 读取现有 README.md
   - 在「原始资料」表格中追加新文件条目
   - 如果新文件中有重要信息（新产品、新客户、新进展），更新相应档案字段

### Step 6：更新关联数据

读取新文件内容，将相关信息直接写入公司 README.md：

1. **人物信息**：文件中提到的关键人物，更新到对应公司 README.md 的「关键联系人」字段
2. **项目信息**：文件中提到的项目/合作，更新到对应公司 README.md 的「相关项目」字段

**注意**：只在有明确新信息时才更新，不要过度推断。对不确定的信息向用户确认。

### Step 7：更新向量索引

调用 MCP 工具 `index_knowledge()` 重建向量索引，确保新归档的内容可被语义搜索。

### Step 8：清理与汇报

1. 确认 `inbox/files/` 已清空（或只剩无法处理的文件）
2. 向用户汇报：
   - 归档了多少个文件，分别归入了哪些公司
   - 新建了哪些公司目录
   - 转换了哪些格式（docx → md）
   - 向量索引已更新

## 关键规则

- **不保存 .docx/.doc 文件**：必须转为 .md
- **会议纪要类文件**：如果内容是会议记录，应归入 `meetings/` 而非公司目录，按 `YYYY-MM-DD-标题.md` 命名
- **先确认后执行**：分组结果和关键更新需用户确认
- **最小化更新**：只更新有明确新信息的字段，不重写整个文件
- **保持一致性**：文件命名、格式、档案结构与现有知识库保持统一
