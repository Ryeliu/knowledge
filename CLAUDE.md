# 知识库操作规则

## 身份
你是我的个人知识管理助手。这个目录是我的本地知识库。

## 文件说明
- `companies/`：公司档案（每家公司一个目录，README.md 为档案入口，含人物和项目信息）
- `meetings/`：按日期命名的会议记录
- `people.md`：人物档案汇总（**已退役，仅供参考，不再更新**）
- `projects.md`：项目台账汇总（**已退役，仅供参考，不再更新**）
- `graph.json`：关联索引（**已退役，由 ChromaDB 向量库替代**）
- `templates/`：LaTeX 模板，用于生成 PDF
- `output/`：编译好的 PDF 输出目录

## 数据架构

**唯一数据源**：
- `companies/公司名/README.md` — 公司档案（含关键联系人、相关项目）
- `meetings/YYYY-MM-DD-标题.md` — 会议纪要

**索引层**：
- ChromaDB 向量库（`~/worklab/mcp-server/vectordb/`）— 自动索引所有 md 文件
- MCP 工具 `index_knowledge()` 重建索引，`search_knowledge()` 语义搜索

## 文件格式规范
- 文本内容统一使用 **md** 或 **LaTeX** 格式，**不保存 .docx 文件**
- 收到 .docx 文件时，提取内容转为 .md 后删除原始 docx
- PDF、xlsx、xmind 等非文本格式可保留原样

## 核心规则

### 录入新信息时
1. 解析输入，识别涉及的公司、人物、项目、会议
2. **公司**：检查 companies/ 中是否已有该公司
   - **已有** → 更新 README.md（新增关键联系人、新项目、新进展）
   - **新公司** → 创建 `companies/公司名/` 目录和 README.md
3. **人物**：写入对应公司 README.md 的「关键联系人」字段
4. **项目**：写入对应公司 README.md 的「相关项目」字段
5. **会议**：在 meetings/ 下新建会议文件
6. **更新向量索引**：调用 MCP 工具 `index_knowledge()` 重建索引

**原则**：录入信息后只需更新 company README.md 和/或 meeting md，然后调用 `index_knowledge()`。不需要更新 people.md、projects.md、graph.json。

### 查询时
1. 调用 MCP 工具 `search_knowledge(query)` 进行语义搜索
2. 根据搜索结果，读取相关的具体文件获取详情
3. 综合输出，不要遗漏关联信息

### 生成 PDF 时
1. 从知识库提取相关内容
2. 填充到对应的 LaTeX 模板
3. 运行 `bash ~/worklab/Sidejob/knowledge/scripts/build.sh <tex文件路径>`
4. 告知用户 PDF 已生成在 output/ 目录

每家公司一个目录，README.md 作为档案入口，包含人物和项目信息。

## 会议记录格式（meetings/YYYY-MM-DD-标题.md）

> **所有会议纪要统一存放在 `meetings/` 目录**，不存放在公司目录中。

```
# 会议标题

- **日期**：
- **参与人**：[姓名A, 姓名B]
- **关联项目**：[项目名]

## 讨论要点

## 决策 / 结论

## 待办事项
- [ ] 谁 负责 什么 （截止日期）

## 下次跟进
```

## 常用指令映射

| 我说的话 | 你要做的事 |
|---------|-----------|
| "录入：……" | 解析并写入知识库，调用 index_knowledge() |
| "会前简报 [项目名]" | 提取相关内容，生成 briefing PDF |
| "项目报告 [项目名]" | 生成项目状态 PDF |
| "查 [公司名/人名/项目名]" | 调用 search_knowledge()，读取详情文件输出 |
| "待办汇总" | 汇总所有文件中未完成的待办事项 |
| "本周回顾" | 汇总本周的会议、新联系人、项目进展 |
